## 현재 상태

## 저장 방법 (페이지당 10초)
1. 브라우저에서 해당 URL 접속
2. `Ctrl+S` → 형식: **"웹페이지, HTML만"** (전체 저장 아님 — 단일 .html이면 충분)
3. 파일명은 아무거나 상관없음 (처리 시 위 표대로 이름을 맞춰줌)
4. 10개 파일을 이 대화에 업로드

> Claude in Chrome을 쓰면 순회 저장을 시킬 수도 있으나, 그 경우 이번처럼
> 오버레이가 저장본에 박힌다 — 어차피 파이프라인이 자동 제거하므로 무방.

## 업로드 후 자동 처리 내용 (페이지당 공통)
- Claude 브라우징 오버레이 제거 (테두리·Stop 버튼·유령 커서·애니메이션 CSS)
- 오류 153 유발 배경영상 iframe 제거 → 배경이미지 대체
- 로컬 실행 시 실패하는 서버 통신 무력화 (로그인 안내 ajax, CRM SDK)
- 내부 링크를 로컬 파일명으로 변환 → **11페이지가 서로 클릭 이동되는 로컬 사이트 완성**
- 최종 결과: zip 패키지로 일괄 전달

## 참고: 라이브 사이트의 오류 153
로컬 파일과 무관하게, 실제 사이트의 히어로 영상 오류는 유튜브 영상
(ipqws5T3gEs)의 임베드 차단/비공개 설정이 원인으로 추정된다.
이는 영상 소유자(안콘텐츠랩)만 유튜브 스튜디오에서 해결 가능:
① 공개 상태 확인(비공개 → 미등록/공개) ② "퍼가기 허용" 체크 ③ 저작권 제한 확인
